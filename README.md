# GitHub Presenter

非公開（プライベート）を含むGitHubリポジトリを、社内共有やクライアント向けに
「見せられるカタチ」でプレゼン表示するための、サーバー不要の静的サイトです。

- **一覧画面**: 自分がアクセスできる全リポジトリ（Public/Private）をカード形式で表示。検索・言語/公開範囲での絞り込み・並び替えが可能。
- **詳細画面**: README（Markdownレンダリング）、使用言語の内訳、コントリビューター、直近のコミット履歴などを1ページのプレゼン資料風にまとめて表示。
- **紹介ページ（成果物公開シート）**: 各リポジトリに「作品のタイトル・アプリURL・スクリーンショット（複数枚、ドラッグ/矢印ボタンで並び替え）・発表用メモ（①解決した課題 ②ターゲット ③特徴・工夫 ④苦労した点）」を登録できる編集フォーム。上から順に埋めればそのまま発表原稿になる構成。
- **プレビュー機能**: 登録したスクリーンショットを◀▶で切り替えられるギャラリー表示、またはアプリURLをその場でiframe埋め込みして確認できる「ライブプレビュー」タブ。両方登録した場合はタブで切り替え可能。
- **プレゼンモード**: ボタン一つで文字サイズを拡大し、画面共有・投影向けの見た目に切り替え。

## 使い方

1. `index.html` をブラウザで直接開くか、任意の静的サーバーで配信してください（GitHub Pages にそのまま置くこともできます）。
2. GitHubで Personal Access Token（PAT）を発行します。
   - `Settings → Developer settings → Personal access tokens`
   - 非公開リポジトリを見る場合は **`repo` スコープ**（Fine-grained の場合は対象リポジトリの **Contents: Read-only**）が必要です。
3. トップ画面でPATを入力し「接続する」を押すと、リポジトリ一覧が表示されます。
4. カードをクリックすると、そのリポジトリの詳細プレゼン画面に遷移します。
5. 詳細画面の「紹介ページ」パネルにある「編集する」から、タイトル・URL・スクリーンショット・発表用メモを登録できます（未入力の項目があっても保存できます）。登録した内容はこの端末の `localStorage` に保存され、一覧のカードサムネイルにも反映されます。

## トークンの扱いについて（重要）

- PATは**このブラウザの `localStorage` にのみ**保存され、GitHub以外のどのサーバーにも送信されません（このアプリ自体にバックエンドはありません）。
- 「このブラウザにトークンを保存する」のチェックを外せば保存されず、タブを閉じると消えます。共有PCではチェックを外してください。
- トップバーの「ログアウト」で保存済みトークンをいつでも削除できます。
- 不要になったPATはGitHub側の設定画面からも失効させることを推奨します。
- 「紹介ページ」の内容（タイトル・URL・スクリーンショット・発表用メモ）も同様に、この端末の`localStorage`にのみ保存されます。GitHub上には一切書き込まれず、他の人やこの端末以外のブラウザとは共有されません。スクリーンショットは自動的に縮小・圧縮して保存しますが、大量に登録するとブラウザの保存容量上限に達し保存に失敗することがあります（その場合は画像枚数を減らしてください）。

## ファイル構成

```
アプリプレゼン用ソフト/
├── index.html      画面構造（ログイン / 一覧 / 詳細）
├── css/style.css   ダークテーマのスタイル
├── js/app.js       GitHub API連携・描画ロジック
└── README.md       このファイル
```

## 技術的な補足

- 外部ライブラリやビルドツールは使用していません（Vanilla HTML/CSS/JS）。
- GitHub REST API (`api.github.com`) を直接呼び出しています。
- Markdownレンダリングは軽量な自前パーサーです（見出し・リスト・コードブロック・リンク・画像・強調・引用に対応）。
- APIレート制限は認証済みで5,000回/時間です。
